- 日期：2026-07-12
- 状态：已评审，待实现

## 0. 背景与目标

构建一个 Excel 周报数据分析器，提供 Web 应用：上传本周销售明细 Excel → 自动解析、计算指标、生成交互式图表与多端交付物（Excel、PPT、网页报告），并用 AI 生成文字总结。支持历史数据保存，以便上周对比与近三周趋势。

### 用户与场景
- 使用者：供应链二部（居家布艺）周报撰写人（王凡）。
- 主流程：每周打开网页 → 上传本周 `.xlsx` → 预览全部结果与图表 → 填写采购/下周计划要点 → 一键生成 → 下载 Excel 与 PPT，AI 文案在页面展示并可复制。

### 核心交付物
1. 交互式 Web 应用（上传 / 预览 / 配置 / 生成 / 下载）。
2. 输出 Excel（`xlsxwriter` 原生图表，可编辑）。
3. 输出 PPT（基于模板 `resource/2026年第二十七周周报-王凡.pptx` 填充图片/文本/表格）。
4. AI 总结文字 4 处（prompt 可编辑配置）。
5. SQLite 历史库（上周对比 / 近三周）。

### 非目标（YAGNI）
- 多用户/权限/登录。
- 自动定时抓取（不做文件夹监视，用户主动上传）。
- 各维度上周对比（仅做全表 KPI 的上周对比；维度对比若将来需要，从原始行复算）。

## 1. 架构总览

### 1.1 技术栈
| 层 | 选型 |
|---|---|
| 后端 | Python 3.12 + FastAPI |
| 前端 | Vue 3 + Vite + Plotly.js（交互图表） |
| 存储 | SQLite（`data/wfreport.db`） |
| 图表 | Plotly（权威 figure / 网页交互）→ kaleido 导出 PNG（供 PPT）；Excel 用 `xlsxwriter` 原生图表 |
| 解析/分析 | openpyxl 读取 + pandas 分析 |
| PPT | python-pptx（模板填充） |
| AI | OpenAI 兼容协议调用 MiniMax-M3；prompt 模板放 YAML |

> 图表引擎决策：网页用 Plotly 交互图；PPT 用同一 Plotly figure 经 kaleido 导出 PNG；Excel 用 `xlsxwriter` 原生图表（可编辑）；热力图等 Excel 不支持的类型用 PNG。一套 Python 图表逻辑为主，Excel 单独原生。

      │                              pandas 计算指标(加权聚合)
      │                                      │
      │                  ┌───────────────────┼──────────────────┐
      │                  ▼                   ▼                   ▼
      │          存入 SQLite 历史      生成 Plotly figure    读取历史(上周/近三周)
      │                                      │

      │              ▼                       ▼                  ▼
      │      kaleido→PNG(嵌PPT)      Excel原生图表         Vue页面交互展示
      │                                      │

## 2. 数据模型与计算公式

> 全部用源列名作为公式变量。

### 2.1 源列（17 列，本周/上周结构一致）
`线上订单号, 品牌, 店铺, 订单日期, 商品名称, 工厂, 商品工厂, 商品编码, 成本价, 标费运费, 平台扣点, 总成本, 销售数量, 销售金额, 销售毛利, 销售毛利率, 是否新品`

### 2.2 派生列
- `平台` = `店铺` 按 `-` 切分取第 0 段（如 `淘宝天猫-猫人严选旗舰店` → `淘宝天猫`）。
- 不在白名单或无 `-` 的前缀（如 `初医生旅行用品`、`小红书`）→ 归入 `其他`。白名单维护在 `config/platform_map.yaml`（可编辑）。
- `周标识(week_id)` = 由 `订单日期` 推导的 ISO 周标识（`YYYY-Www`，如 `2026-W27`）。

### 2.3 通用聚合规则（关键）
任意维度 D（品牌 / 平台 / 店铺 / 工厂 / 商品编码 / 订单日期 …）下：
- `销售额(D)` = `Σ 销售金额`（按 D 分组求和）
- `销售数量(D)` = `Σ 销售数量`
- `销售毛利(D)` = `Σ 销售毛利`
- `销售毛利率(D)` = `销售毛利(D) / 销售金额(D)` ← **加权毛利率**
- `销售毛利率(D)百分比` = `销售毛利率(D) × 100`，保留 2 位小数

> 源表有逐行 `销售毛利率`，但聚合时**不使用**（行级均值错误，如本周行均值 49.56%，加权法 50.1%）。一律用 `Σ销售毛利 / Σ销售金额`。已用样例验证：865017.09 / 1726680.2 = 50.1%，与模板一致。

### 2.4 对比与排名公式
- `环比变化量(指标, D)` = `指标(本周, D) − 指标(上周, D)`
- `环比变化率(指标, D)` = `(指标(本周, D) − 指标(上周, D)) / 指标(上周, D)`
- `销售数量占比(D)` = `销售数量(D) / Σ全部 销售数量`
- `销售金额占比(D)` = `销售额(D) / Σ全部 销售金额`
- `TOP-N(D)` = 按 `销售数量(D)` 降序取前 N（店铺/产品/工厂场景统一按销售数量排名）
- `最近三周` = 从 SQLite 取最近 3 个 `week_id` 的 `销售额 / 销售毛利 / 销售毛利率`

### 2.5 各模块维度
| 模块 | 维度 D | 关键公式 |
|---|---|---|
| 汇总 | 全表(本周) | `Σ销售金额`、`Σ销售毛利`、`销售毛利/销售金额` |
| 每日 | `订单日期` | 每日 `Σ销售金额`、每日 `Σ销售毛利/Σ销售金额` |
| 品牌 | `品牌` | `销售额/销售数量/毛利率(品牌)`、`销售数量(品牌)/Σ全部销售数量` |
| 平台 | `平台`(派生) | `销售数量(平台)`、`销售毛利(平台)/销售金额(平台)` |
| 店铺 | `店铺` | TOP15(按`Σ销售数量`) → 每日 `Σ销售数量` |
| 产品 | `商品编码`(+`商品名称`) | TOP15 `销售数量/毛利率`；每日趋势 |
| 新品 | `是否新品=='是'` 的 `商品编码` | `销售数量/毛利率` |
| 工厂 | `工厂` | `Σ销售数量` TOP5 |

## 3. 各分析模块与图表细节

通用图表要求：美观、无文字/数值重叠、显示 Y 轴具体数值或数据标签、配色统一。双轴图统一：左轴量纲、右轴毛利率；需求数值用数据标签而非坐标轴刻度。

### 3.1 汇总（PPT 第 3 页）
- **① 概览组合图**：x 轴 = `本周`（单类目）；`Σ销售金额`（柱，仅数据标签、不显示坐标轴刻度）+ `Σ销售毛利`（柱，左轴）+ `销售毛利率`（折线/标记点，右轴）。三个指标均显示具体值。
- **② AI 文本（上周对比）**：输入 `本周/上周 {销售额,销售毛利,销售毛利率}` + 环比量/率 → AI 输出对比叙述。填入 PPT 左侧文本框。
- **③ 最近三周表格**：3 行 ×（周标识、销售额、销售毛利、销售毛利率）。PPT 渲染为 PNG 嵌入右侧；网页/Excel 用真表格。

### 3.2 每日销售趋势（PPT 第 4 页）
- **组合图**：x 轴 = `订单日期`(7 天)；左轴 `每日 Σ销售金额`（柱，显示数值）+ 右轴 `每日 Σ销售毛利/Σ销售金额`（折线，显示数值）。
- （需求另提"所有产品每日毛利率"与上图为同一口径，合并为一条折线，不重复出图。）
- **AI 总结**：输入每日序列 → AI 概括趋势（峰谷、周末效应等）。

### 3.3 品牌分析（PPT 第 5、6 页）
- **第 5 页组合图**：x 轴 = `品牌`(猫人/初医生)；`销售额`(柱，仅标签) + `销售数量`(柱，左轴) + `毛利率`(折线，右轴)。
- **第 6 页饼图**：各品牌 `销售数量(品牌) / Σ全部销售数量` 占比。
- 品牌叙述文本（PPT 第 5 页文本框）：**程序按模板拼接**（不调 AI）：每品牌"销售额/数量/毛利率" + 对比结论。

### 3.4 平台分析（PPT 第 8 页）
- 需求要①扇形占比 ②数量+毛利率双轴。模板第 8 页仅 1 图位 → 用 **1×2 子图合成一张 PNG**：左饼（各平台 `销售数量(平台)/Σ全部` 占比）+ 右双轴（`销售数量`柱左轴 + `毛利率`折线右轴）。

### 3.5 店铺分析（PPT 第 7 页）—— 用热力图解决"15 条线"
- TOP15 店铺（按 `Σ销售数量` 降序）。
- **热力图（PNG）**：行 = TOP15 店铺（横向标签，避免拥挤）、列 = 7 天日期、颜色深浅 = `每日 Σ销售数量`，格内显示数值。一张图读出每店每天量级与趋势，替代 15 条折线。

### 3.6 产品分析（PPT 第 9 页 + 复制页 9'）—— 同样用热力图
- **① TOP15 产品（按 `Σ销售数量`）组合图**：`销售数量`柱(左轴) + `毛利率`折线(右轴)，x 轴为 `商品名称`（名称过长截断 + tooltip 全名）。
- **② TOP15 产品每日趋势热力图**：行 = TOP15 产品、列 = 7 天、色 = `每日 Σ销售数量`。
- **③ 新品图（`是否新品=='是'`）**：x 轴 = 新品 `商品名称`；`销售数量`柱(左轴) + `毛利率`折线(右轴)。
- **布局决策（已定）**：复制第 9 页得到两页：
  - **第 9 页**：文本框←产品叙述；图←①TOP15 组合图 + ②每日热力图（1×2 子图 PNG）。
  - **第 9' 页（复制）**：图←③新品组合图；文本←新品叙述。

### 3.7 供应商（工厂）分析（PPT 第 10 页）
- 表格：`工厂` 按 `Σ销售数量` 降序取 **TOP5**，列出 `Σ销售数量`（附 `Σ销售金额`、`销售毛利率`）。
- PPT 第 10 页是图片位 → 表格渲染为 PNG 嵌入；Excel/网页用真表格。

### 3.8 采购情况（PPT 第 12–14 页，3 表，自动分页）
- 用户在网页文本框输入简要采购工作要点 → AI 润色、分条、结构化。
- 模板采购表 5 列：`序号 / 事项内容 / 进度及责任人 / 状态 / 完成时间`（"进度及责任人"为多行子要点）。
- **自动分页机制**：
  1. AI 输出结构化 JSON 数组：`[{事项内容, 进度及责任人:[子要点…], 状态, 完成时间}, …]`；prompt 约束子要点 ≤6 以控高。
  2. 按可配置 `每页条数`（默认 4）切片：`chunks = [items[i:i+n] for i in range(0,len(items),n)]`。
  3. 依次填第 12→13→14 页模板表格；`序号` 跨页自增；`进度及责任人` 用换行连接子要点；行数不匹配用 XML 增删 `<a:tr>`。
  4. 溢出（>3 页容量）→ 复制采购页（XML deepcopy）继续放；不足 → 多余页留表头空表或删除。

### 3.9 下周计划（PPT 第 16 页，1 表）
- 用户在网页输入简要下周计划 → AI 润色分条 → 填表格。
- 模板表 5 列：`序号 / 事项内容 / 提出时间 / 次周预计完成节点名称 / 涉及部门`，10 行容量；溢出同理复制。

## 4. 交付物细节

### 4.1 SQLite 历史库（`data/wfreport.db`）
两张表：
- **`weekly_summary`**（必需，驱动上周/近三周）
  `week_id(PK), 周起始日, 周结束日, 销售额, 销售毛利, 销售毛利率, 销售数量`
- **`weekly_orders`**（原始行归档，保险+未来任意维度对比）
  `id, week_id(FK), 线上订单号, 品牌, 店铺, 平台, 订单日期, 商品名称, 工厂, 商品编码, 成本价, 总成本, 销售数量, 销售金额, 销售毛利, 销售毛利率, 是否新品`

`week_id` 去重：上传同周数据 → 提示"已存在，是否覆盖"。上周对比 = `week_id < 当前` 的最近 1 条；近三周 = 最近 3 条。

### 4.2 Excel 输出（`xlsxwriter` 原生图表）
一个工作簿，多 sheet：
- `汇总`：本周 KPI（销售额/毛利/毛利率）+ 近三周表 + 原生组合图（柱+柱+折线双轴）。
- `每日`：每日明细表 + 原生双轴图（销售额柱 + 毛利率折线）。
- `品牌`：品牌表 + 原生组合图 + 占比饼图。
- `平台`：平台表 + 饼图 + 双轴图。
- `店铺`：TOP15 每日透视表 + 热力图 PNG（Excel 原生不支持）。
- `产品`：TOP15 表 + 新品表 + 原生双轴图 + 热力图 PNG。
- `工厂`：TOP5 表。
- `AI文案`：4 段 AI 输出文字。
- 统一：表头样式、冻结首行、金额格式 `#,##0.00`、毛利率格式 `0.00%`。

### 4.4 Web 应用（FastAPI + Vue 3）
**页面：**
- **上传页**：拖拽上传本周 `.xlsx`；解析校验（行数/日期范围/列完整性）；确认 `week_id`。
- **预览页**：分区块展示全部计算结果与 Plotly 交互图（汇总/每日/品牌/平台/店铺/产品/新品/工厂），每块附数据表；右侧上周对比/近三周。
- **AI 文案区**：4 处可单独"重新生成"；采购/下周计划各一个大文本框（输入简要 → AI 润色 → 可编辑结果）。
- **配置页**：在线编辑 `prompts.yaml`（4 模板）+ `platform_map.yaml`；改完热加载。
- **生成/下载页**：一键生成 → 下载 Excel、PPT；报告网页可独立保存。

**API（REST）：**
- `POST /api/upload` 上传 → 返回 `week_id` + 解析校验
- `POST /api/preview/{week_id}` 返回全部分析结果 JSON（含 Plotly figure 配置）
- `GET /api/history` 近三周/上周
- `POST /api/ai/{section}` section ∈ {week_compare, daily_summary, procurement, next_plan} → AI 文本
- `GET/PUT /api/config/prompts`、`/api/config/platform` 读写配置
- `POST /api/generate/{week_id}` 生成 Excel+PPT，返回下载链接
- `GET /api/download/{file}` 下载产物

**配置文件：**
- `config/prompts.yaml`：4 个 prompt 模板（含 `{{变量}}` 占位）。
- `config/platform_map.yaml`：店铺前缀→平台映射白名单。
- `config/settings.yaml`：AI 的 base_url/model；`api_key` 放 `.env`（加入 `.gitignore`，不进 git）。

## 5. AI Prompt 模板与配置

**调用**：OpenAI 兼容协议 → `https://api.minimaxi.com/v1`，model `MiniMax-M3`，api_key 读 `.env`。封装 `ai/client.py`（统一超时/重试/错误兜底，失败回退程序拼接占位文本并在前端标红）。

**4 个 prompt 模板**（`config/prompts.yaml`，可在线编辑、热加载），变量用 `{{双花括号}}`：
- **`week_compare`（上周对比，§3.1）**：输入 `本周/上周 {销售额,销售毛利,销售毛利率}` 及环比量/率；输出 2–4 句中文对比叙述。
- **`daily_summary`（每日趋势总结，§3.2）**：输入 7 天 `销售额/毛利率` 序列；输出趋势概括（峰值日、低谷日、周末效应等），分条 ≤5 点。
- **`procurement`（采购情况，§3.8）**：输入用户自由文本要点；输出结构化 JSON 数组（`事项内容/进度及责任人/状态/完成时间`），约束子要点 ≤6。
- **`next_plan`（下周计划，§3.9）**：输入用户自由文本要点；输出结构化 JSON 数组（`事项内容/提出时间/次周预计完成节点/涉及部门`）。

每个模板含 `system`（角色：供应链周报撰写助手，口径务实简洁）+ `user`（含变量）两段。配置页可改 `system/user`、变量占位、温度等；保存即生效。

**安全**：api_key 只放 `.env`（加入 `.gitignore`），不进代码/文档/git。

## 6. 质量要求与验收
- 所有图表美观、无文字/数值重叠、显示 Y 轴数值或数据标签；双轴图按要求分配量纲。
- 全部聚合指标用加权法（`Σ销售毛利/Σ销售金额`），不用行级均值。
- PPT 产物基于模板，品牌视觉一致；图片不变形；表格自动分页正确、序号连续。
- Excel 产物图表为原生可编辑对象（热力图除外）。
- AI 调用失败有兜底，不阻断生成。
- 历史库正确支持上周对比与近三周。
